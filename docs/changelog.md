### 印章新增十三人（首次出现，非替换）

此前接入的均为「替换」——人物本有 `pin`（一字品评），文字印章已在显示，改挂 SVG 而已。本轮十三人**原 `pin` 为空、本不显印章**，是首次出现印章，故性质是新增。分四批：程凯（烈）、齐晟（果）；程锋（景）、李庭深（穆）、程铎（严）、程钊（贞）；李庭泽（怀）、杜徵（皎）、岑菡（雅）、卢秉毓（思）；许秋生（翼）、景惠时（懿）、魏谦（捷）。累计 **29 人**走 SVG 印章。

- **改动两处联动，缺一即错**：`Seal` 取字自 `c.pin`，`pin` 又是详情页印章的显示开关（`c.pin && …`），空值时列表与详情都不出印章。故除在 `seals.js` 登记外，须在 `cangyun-data.js` 给这十三人补 `pin`。二者必须同改——只补 `pin` 不登记会冒出文字印章，只登记不补 `pin` 则 `Seal` 直接返回 `null`、`check-data` 报错。
- **`pin` 的两个副作用均属预期**：一是这十三人从此可按品评字（如「懿」「捷」）站内检索，与既有诸人一致；二是它作为隐藏字撑起 SVG 占位盒，但任一 CJK 字等宽等高，故不影响排版，`SEAL_DENSE_CH` 五个繁复字亦无一在内。
- **品评字来源与一处确认**：多数由上传时直接给定。程锋一批四人当时只随文件名附了拼音（jing／mu／yan／zhen），而一字品评系作者所定、且落为**不可见的隐藏字**（页面只见印面图，看不见 `pin`），事后无从目测校验，故逐字确认后方落库——景／穆／严／贞。其中 jing 依「镇梁州御京师」本欲取「靖」，实为「景」，幸先问过。
- **文件名与 id 不同形，再添一例**：许秋生 id 为 `xuqiusheng`，印章文件却名 `xvqiusheng-yi.svg`（拼音以 v 代 ü），故映射表显式列举、不由 id 拼路径；`check-data` 比对文件是否存在兜底。
- 十三张图红面路径（`class="st0"` 的 `d`）逐字节同模板，沿用小印 1.15、大印 1.20，未逐张调参。

### check-data 增印章图校验

映射表指向的文件若被改名或删除，浏览器只渲染一枚破图、构建不报错——本次接入途中恰好遇上一次改名（`lifeichenglie-ba.svg` → `lifei-ba.svg`），遂补此节，比对映射表、人物表与 `public/images/seals/` 三方。

- **报错（挡下 `npm run lint`）**：文件不存在；键非有效人物 id（该人物会静默退回文字印章）；该人物无 `pin`（`Seal` 直接返回 `null`，印章根本不出现）。
- **报警告**：目录中无人引用的孤儿图，提示「上传后忘了写进 `SEAL_SVG_FILE`？」。定为警告而非错，因批量上传、逐个接入途中目录里必然有尚未登记的图，报错会挡住正常流程。
- 改名一类会同时报出错与警告——错指出映射表里的死引用，警告指出那张变成孤儿的真文件，新旧文件名并排可见。
- 五种情形均以人为制造故障实测触发过，非只写不验。

### 三图触屏适配（`01132bc`）

行星图、群像图、舆地图此前只认鼠标，手机上不可用。三图各自一套缩放平移代码，此次抽出共用的 `usePanZoom(w, h)` 与 `ZoomBtns` 一并接入。

- **双指捏合**：多指以 `Map<pointerId>` 记账、`setPointerCapture` 接管，缩放以两指中点为锚。此前第二根手指落下会无条件覆写拖曳原点，双指操作时图幅跳变。
- **1× 时不夺页面滚动**：`touchAction` 仍是 `k>1` 才置 `none`。单指可照常划过图幅翻页，放大后才吃手势。代价是捏合放大须先按 ＋，换来的是整幅图不至于成为手机上的滚动死区。
- **悬停专属信息改点选可及**：触屏无悬停，此前有几处信息在手机上根本够不着。按 `useCoarsePointer()` 分流，桌面行为一概不变——
  - 群像图、行星图：轻触人物先显其关系边（弧），再触方开档案
  - 舆地图动线段：轻触浮出「繫此段之事」，且不转开档案（档案由驻点与人名承接）；浮框改浮于指上方，免为指腹所掩
  - 各图另加透明衬底，点空处即卸下所选
- **触屏点选域加大**：群像节点、行星节点、地点环、动线感应带在窄屏上实际只有几像素宽。
- **行星图补上缩放平移**：此前完全没有，1200×830 的幅面直接挤进手机宽度。

顺带修掉两个**既有缺陷，桌面同样中招**：

1. `movedRef`（防拖后误触的位移量）只在 `pointerdown` 里重置，而该回调在 `k===1` 时提前 return。于是「放大 → 拖动 → 按『回』复位」之后它永停在 >5，此后**一切点选**被当作拖后误触吞掉，只能刷新页面。
2. `setView` 的 updater 延后解引用 `dragRef.current`。React 19 中 `pointermove` 属连续事件被排队延后，`pointerup` 属离散事件抢先同步跑完并把 `dragRef` 置空，updater 这才执行、读 `null.tx` 抛错，整棵 React 树卸载。桌面鼠标 move 与 up 间隔大几乎撞不上，**安卓上一抬手就白屏**。改为先取快照、updater 只用局部值。

另在 `src/main.jsx` 加了开发期错误浮层（`import.meta.env.DEV` 守着，不入打包）：未捕获错误直接印在页面底部，手机上无控制台时靠它取堆栈。

### 门派纹三图再瘦身（`8c69152`）
