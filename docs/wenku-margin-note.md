# 文库页边批注 · 增补工作流

> 文库阅读页「角色名点击 → 页边批注卡」功能的维护文档。
> 现状：全量接入（2026-07-18）——词条以本名自 CHARACTERS 自动派生，
> 全部人物在正文中即点即卡；`readerNote` 简介文案按人逐步补写中。

## 功能概览

正文中的角色名默认与普通正文无异，hover / 键盘聚焦 / 批注打开时显示门派色 1px 虚线下划线；点击（或 Tab 聚焦后 Enter/Space）在正文栏右侧批注区弹出小卡，含门派色虚线引导线与卡缘圆点，视觉如书籍眉批。

涉及文件：

| 文件 | 职责 |
|---|---|
| `src/cangyun-data.js` | 人物数据，含批注专用字段 `readerNote` |
| `src/novel/char-terms.js` | 词条表（`CHAR_TERMS`）与识别逻辑（`scanCharTerms`，纯函数、可 Node 测试） |
| `src/novel-reader.jsx` | 命中段渲染（`splitCharTerms`）、批注卡（`MarginNote`）、引导线 |

## 维护方式（全量接入后）

### 识别层：无需日常维护

词条自 CHARACTERS 全量自动派生（`src/novel/char-terms.js`），新人物入库即自动可点。门派强调色自动经 `TERM_ACCENT` 从该人物 `belong[0]` + `fc()` 派生。识别为多词条遍历：每步取最早命中处、等位取最长词（长名不被短名吞掉）。

只在两种冲突出现时才需要动手，方式都是把 id 加入 `TERM_EXCLUDE` 摘除自动词条：

1. **同名人物入库**：`npm run lint`（check-data）会以「term 重复」报错拦下——摘除后另行人工映射（映射方案届时再定，不能只靠字符串匹配）；
2. **人名恰为常语**（正文出现「关山难越」之类非指人的用法）：摘除该 id，或按需改为手工条目。

### 数据层：readerNote 简介（作者编辑的部分）

在目标人物对象中加顶层可选字段 `readerNote`，惯例放在 `gloss` 与 `profile` 之间（参照 gaoyuan 条目）：

```js
readerNote: "凌雪阁机枢府要员。",
```

规则：

- 有 `readerNote` → 批注卡**原样显示、不截断**（长度作者自控，卡片有 maxHeight + 内滚兜底）；
- 无此字段 → 自动回退 `profile[0][1]`（F() 第一组「官职身份」）的 110 字截断摘要——全量接入后这是大多数人物的现状，**回退即可用，不必赶工补写**；
- 建议单条 60–120 个汉字；
- 不动 `F()`；完整档案页 DetailPanel 不读取此字段，完全不受影响。

### 无需改动的部分

`MarginNote` 卡片对所有人物通用：显示 name / 字 / 号 / 门派 chip / 生卒 + 籍贯 + 批注文案，无按钮，接入即生效。

## 实现约定（改动时的红线）

- **识别**：只在纯文本节点内精确匹配、拆分文本节点渲染，不做 HTML 字符串 replace；检索高亮命中段内不识别角色名。
- **排版**：不重排正文、不改正文宽度（`bodyMaxWidth` 714）、不锁 body 滚动、无全屏遮罩。
- **定位三档降级**（以 `room = 视口宽 − 12 − 正文右边界` 为判据，卡宽 260）：
  1. `room ≥ 272`：完整旁注位，卡置正文右边界 + 12px；
  2. `200 ≤ room < 272`：吸附视口右缘，允许覆盖正文最右 ≤ 60px；
  3. `room < 200`：退为贴名字上/下方的 popover，无引导线。
- **关闭**：Esc / 点卡外（`.nvl-char` 豁免 = 移位重开）/ 页面滚动逾 24px / resize / 换章 `jump()`。
- **引导线与圆点**：`noteInk()` = 门派色向白提亮 25%、alpha 0.5（素纸主题下改门派原色、alpha 0.7——深底提白在纸上会沉没）；虚线 3px 划 / 3px 空、1px 高，位于名字下划线延长高度。
- **下划线**：`.nvl-char` 虚线常备、颜色透明，三种激活态仅变色为 `--char-accent`，无版式抖动。

## 人工编辑注意

- 只修改 `readerNote` 双引号内的文本；
- 行尾逗号必须保留；
- 文本中若需要使用英文双引号，应写成 `\"`，或改用中文引号；
- 保存后由 Vite 自动热更新；
- 若页面报错，优先检查是否漏掉引号、逗号或括号；
- 建议单条旁批控制在 60–120 个汉字。
